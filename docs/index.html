Detecting Distribution Shift
in Lane Perception Models

A two-sample statistical testing framework that flags when a lane detection model encounters out-of-distribution inputs — no target-domain labels required.

GitHub 🚀 Command Generator
Methodology

Pipeline

Three sequential stages estimate a calibrated threshold and report shift detection rates across independent runs.

Step 1 — Extract Source Features
A fixed Source Set and many randomly-sampled calibration sets are drawn from the source domain and encoded into compact latent vectors by a ResNet-18 encoder — using ImageNet, randomly initialized, or custom-trained weights. Two-sample statistical tests (MMD, Energy, BKS, MMD-Agg) are then computed between the source and each calibration set. Encodings are cached to disk under Encodings/dim_<N>/ so repeated experiments are instant.
Step 2 — Calibration
The statistical test results from the calibration sets are sorted to form a null distribution for each test. The threshold τ is set at the (1−α) percentile of that null distribution — e.g., α = 0.05 keeps the false-positive rate at 5 % on same-domain data.
Step 3 — Shift Test
The same Source Feature Set from Step 1 is compared against randomly sampled sets from the target domain using the same statistical tests. If a test statistic exceeds τ, a shift is detected for that run. The True Positive Rate (TPR) is reported across all independent runs.
Evaluation

Results

True Positive Rate (TPR) for each statistical test across all dataset pairs (100 runs, α = 0.05). The lane detector F1 score serves as an independent proxy for domain distance — high TPR should correlate with low F1.
