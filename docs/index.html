Methodology

Pipeline

This methodology provides a robust, target-label-free way to detect when a lane perception model encounters out-of-distribution (OOD) data. By leveraging a ResNet-18 encoder and two-sample statistical tests, the pipeline extracts source features, calibrates detection thresholds based on same-domain data, and evaluates dataset shifts in real-time across independent runs.

Step 1 — Extract Source Features
A fixed Source Set and many randomly-sampled calibration sets are drawn from the source domain and encoded into compact latent vectors by a ResNet-18 encoder. Two-sample statistical tests (MMD, Energy, BKS, MMD-Agg) are computed between the source and calibration sets. Encodings are cached to disk so repeated experiments are instant.
Step 2 — Calibration
The statistical test results from the calibration sets are sorted to form a null distribution for each test. The threshold τ is set at the (1−α) percentile of that null distribution — e.g., α = 0.05 keeps the false-positive rate at exactly 5% on same-domain data.
Step 3 — Shift Test
The same Source Feature Set is compared against randomly sampled sets from the target domain. If a test statistic exceeds τ, a shift is detected for that run. The True Positive Rate (TPR) is calculated and reported across all independent runs.

Results

True Positive Rate (TPR) for each statistical test across all dataset pairs (100 runs, α = 0.05). The lane detector F1 score serves as an independent proxy for domain distance — high TPR should correlate with low F1.
